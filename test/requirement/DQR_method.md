# PEF method
数据收集要求和质量要求
公司特定数据
本节阐述公司特定生命周期清单（LCI）数据的相关要求：此类数据需在特定设施或一组设施内直接测量或收集，且需能够代表系统边界内的一项或多项活动/过程。
数据需涵盖该过程所有已知的输入与输出。输入示例包括：能源、水、土地的使用及材料消耗；输出示例包括：产品、副产品、各类排放物及产生的废物。其中，排放物需按排放去向分为三类，即向空气、水和土壤的排放。
公司特定排放数据的收集方法包括多种：既可基于直接测量获取，也可通过公司特定活动数据结合相关排放因子（如燃料消耗量（单位：升）与车辆或锅炉燃烧排放因子）计算得出。若范围内的产品所属部门受欧盟排放交易体系（EU ETS）监测规则约束，PEF 方法使用者需遵循《法规 (EU) 2018/2066》中针对所涵盖过程及温室气体的量化要求；关于碳捕获与封存，需以本附件规定为准。数据可能需经过缩放、聚合或其他形式的数学处理，确保其与该过程的功能单元及参考流相匹配。
公司特定数据的典型具体来源有：
	工艺或工厂级消耗数据；
	消耗品账单及存货（库存）变动数据；
	排放测量数据（含烟道气、废水的排放量与浓度）；
	产品与废物的组成数据；
	采购及销售部门/单位提供的数据。
开展 PEF 研究时创建的所有新数据集，均需符合 PEF 标准要求。
所有公司特定数据需在公司特定数据集中完成建模。
物料清单（BoM）分为两部分：一是物料/配料清单，二是每种物料/配料的用量。
BoM 的活动数据需与产品范围对应，且需采用公司特定数据建模。对于生产多种产品的企业，所使用的活动数据（含 BoM）须针对研究覆盖的产品确定。
制造工艺的建模需以公司特定数据为基础（例如，组装研究范围内产品的材料/组件所需的能量）。对于生产多种产品的企业，所使用的活动数据（含 BoM）须针对研究覆盖的产品确定。
次级数据
次级数据是指非基于直接测量或系统边界中各自过程计算的数据。次级数据要么是特定行业的，即特定于 PEF 研究所考虑的行业，要么是多行业的。次级数据的例子包括：
	文献或科学论文中的数据；
	来自 LCI 数据库、行业协会报告、政府统计等的行业平均生命周期数据。
所有次级数据均应纳入次级数据集进行建模，且该次级数据集需符合第 4.6.3 节的“数据层级要求”以及第 4.6.5 节规定的“质量要求”。所用这些数据的来源应清晰记录，并在 PEF 报告中予以披露。
使用的数据集
PEF 研究中，若存在符合 EF 合规要求的次级数据集，应优先使用。如需构建符合 EF 合规要求的次级数据集，需遵循《EF 合规数据集指南》（详见https://eplca.jrc.ec.europa.eu/permalink/Guide_EF_DATA.pdf）。不存在符合 EF 合规要求的次级数据集，或无法构建该类数据集，则应按照以下按层级顺序规定的规则，选择待使用的数据集。
	优先使用符合 EF 标准的代理数据集（若有）；代理数据集的使用需在 PEF 报告的“限制”部分说明，且从过往 EF 合规系统（如从 EF 2.0 转换至 EF 3.0）获取的数据，均视作代理数据。
	可使用 ILCD 入口级（EL）兼容的数据集作为代理（在使用 ILCD-EL 兼容数据集的情况下，基本流的命名应与模型其余部分中 EF 兼容数据集所使用的 EF 参考包保持一致，EF 开发人员页面上的以下链接获得：http://eplca.jrc.ec.europa.eu/LCDN/developerEF.xhtml）；单一总分中，来自 ILCD-EL 兼容数据集的占比不得超过 10%。
	若不存在符合 EF 标准或 ILCD-EL 标准的数据集，需将对应过程排除在模型之外。此情况需在 PEF 报告的“限制”部分作为数据缺口明确说明，且需经验证方验证。
截断
除非符合下述规定，否则应避免任何截断操作。
依据物质与能量流及环境重要性水平（单一总分），过程与基本流累计最多可排除 3.0%。在环境影响评估报告中，需明确且合理说明受限过程，尤其要阐述所采用限制措施的环境意义。
除背景数据集中已包含的截断点外，还需纳入本截断规则的考量。本规则对中间产品与最终产品均适用。
累计占物质与能量流比例不足 3.0% 的过程，及其对各影响类别的环境影响，可排除在 PEF 研究之外。
建议开展筛选研究，以确定可截断的过程。
数据质量要求（DQR）
本节描述如何评估符合EF标准的数据集的数据质量。数据质量要求如表 20所示。
	两项最低质量要求：
	完整性（completeness）：LCI 结果数据集应全面覆盖系统内各过程与产品的资源输入与排放输出，能够支撑所有相关环境影响指标的计算；
	方法学适当性与一致性（methodological appropriateness and consistency）：LCI 结果数据集的建模方法应符合定义的目标与范围要求，保持方法学的一致性，确保数据应用的准确性与可比性。
	四项质量标准：技术、地理、时间代表性和数据精确性。这些标准应遵循评分程序。符合EF标准的数据集指南解释了如何在数据集中报告它们
	三个质量方面：文档、术语和审查。这些标准不包括在数据质量的半定量评估中。EF合规数据集指南解释了如何在数据集中执行和报告这三个质量方面。

企业特定数据集（初级数据）质量评价
企业特定数据集属于部分终止系统数据集，应分别对以下两类型的数据进行的数据质量等级评估：
	企业特定的活动数据；
	企业特定的基本流数据（即直接排放数据）。
应分别对活动数据和基本流数据进行数据质量等级评估。
对公司特定数据集进行部分分解：评估活动数据和直接基本流的 DQR。子过程的 DQR 应通过数据需求矩阵进行评估
新开发的数据集的数据质量等级（DQR）应按照以下步骤计算：
	选取最相关的活动数据和基本流数据。
	最相关的活动数据指关联子过程（即次级数据集）产生的环境影响累计占该企业特定数据集总环境影响至少 80% 的数据。应按照贡献从高到低的顺序列出。
	最相关的基本流指累计贡献至少达到基本总环境影响的 80%。
	对每种最相关活动数据和最相关直接基本流。
依据表 23 分别评估对应的四项 DQR 指。
	每一个最相关的基本流由量值及其命名组成（如40g CO2），需分别评估对应的四项DQR指标（即 TeREF、GeREF、TiREF、PEF）。
	每一最相关活动数据也需评估相应的四项DQR指标（即 TeRAD、GeRAD、TiRAD、PAD）。
	鉴于活动数据和基本流均应为初级数据。P 的评分应小于或等于3分，TeR、TiR 和 GeR 的评分应小于或等于2分，总体DQR分值应小于或等于1.5分。
	计算各最相关活动数据和基本流的环境贡献比例。
分别确定各最相关活动数据（对应子过程）和基本流对总环境影响的相对贡献比例。例如：若新开发数据集中两项最相关活动数据共同贡献 80% 的总环境影响，则：
	活动数据1贡献30%，其相对权重为37.5%（即30% / 80%）
	活动数据2贡献50%，其相对权重为62.5%（即50% / 80%）
	加权计算TeR、TiR、GeR和P指标的平均值。
采用步骤3中得到的权重，对每项DQR指标进行加权平均，分别得到TeR、TiR、GeR和P的加权平均值。
	计算新开发数据集的总DQR分值
总DQR分值应按公式 20 式计算其中，¯("TeR" )、¯("GeR" )、¯("TiR" ) 和 ¯("P" ) 为第4步中加权计算得到的加权平均值。

数据需求矩阵
应使用数据需求矩阵（DNM）对建模目标产品所需的所有过程的数据要求进行评估（见表25）。
数据需求矩阵指明了针对各类过程，应使用企业特定数据还是次级数据，这一选择依据公司对该过程的影响程度确定。数据需求矩阵中区分了以下三种情形，并解释如下：
情形 1：该过程由开展 PEF 研究的公司自身运行。
情形 2：该过程不是由开展 PEF 研究的公司运行，但该公司可以获取（企业）特定信息。
情形 3：该过程既不是由开展 PEF 研究的公司运行，且该公司也无法获取（企业）特定信息。
使用PEF方法的单位应执行以下要求：
	确定公司对供应链中每一个过程的影响程度（属于情形1、2或3）。
	这一判定将决定表25中针对每个过程适用的选项；
	在PEF报告中提供一张表格，列出所有过程及其对应的DNM情形；
	遵循表25中所指示的数据要求；
	按照第4.6.5.6节至第4.6.5.8节的要求，对最相关过程及新开发过程所用数据集的DQR（各单项及总分）进行计算或重新评估。

数据需求矩阵（DNM）情形1
对于所有由企业自身运行的过程，且企业在开展 PEF 研究中使用了企业特定数据的情况，应按照第 4.6.5.2 节的要求，对新开发的 EF 符合性数据集进行 DQR 评估。
数据需求矩阵（DNM）情形2
当某一过程属于情形 2（即该过程并非由开展 PEF 研究的企业运行，但该企业可获取企业特定信息）时，可选择以下两种方案：
	PEF方法使用者可获取供应商的详细信息，具备创建EF符合性数据集的条件（选项1）；
	企业仅掌握部分供应商信息，能够进行最低限度的替代（选项2）。
情形2 / 选项1
对于不由企业自身运行、但使用了企业特定数据的所有过程，应按照第4.6.5.2 节的要求，对新开发的 EF 符合性数据集进行 DQR 评估。
情形2 / 选项2
在情形 2 的选项 2 中，应使用非聚合形式的 EF 符合性次级数据集。开展PEF 研究的企业应满足以下要求：
	使用企业特定的运输活动数据；
	将该非聚合数据集中原有的电力结构与运输过程替换为供应链特定的 EF 符合性数据集。
可使用企业特定的 R1 参数值。PEF方法使用者应对该过程的 DQR 指标重新计算，并通过表24对技术代表性（TeR）与时间代表性（TiR）进行情境化再评估。地理代表性（GeR）指标应在原始分值基础上降低 30%，数据精确度（P）指标保持原值。
数据需求矩阵（DNM）情形3
当某一过程属于情形 3（即该过程既不由企业自身运行，且该企业也无法获得企业特定数据）时，企业应使用EF符合性次级数据集。

若该过程不属于最相关过程，则应直接采用原始数据集中的DQR指标，无需重新评估。
